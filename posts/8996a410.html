Git语句

Git：分布式版本控制系统

1.基本信息

版本查询

git version

基本信息查询

git config -l

基本信息配置

git config --global user.name "用户名"
git config --global user.email "邮箱"

2.创建提交语句

创建文件夹

mkdir jxyy $创建文件夹

初始化仓库

git init

创建文件

touch 文件名

查看文件状态

git status

  • 绿色：暂存区

  • 红色：工作区

添加到暂存区

git add .

提交到版本库

git commit -m "描述"

3.删除操作

工作区删除

rm 文件名

暂存区

git rm 文件夹名

版本库

git commit -m "描述"

4.修改文件名

工作区

mv 旧文件名 新文件名

暂存区

git rm 旧文件名
git add 新文件

5.查看版本和状态

git status
git diff
git log --oneline -n数量

6.状态改变

场景一：修改了工作区，但是发现代码出现问题了！

     git checkout 文件名

     将版本库的对应文件的最新版本，覆盖工作区对应文件！

     文件一定被提交过！！

场景二:错误文件提交到了暂存区！怎么处理！

     危险命令： git reset --hard 

     步骤1：暂存区状态去掉

     git reset head 文件名

     步骤2：工作区覆盖

     git checkout 文件名

场景三：提交到版本库或者远程仓库

减量更新：版本回滚(不推荐)
      git reset --hard

           git reset --hard 版本号

         问题：会去掉别人的代码！

         增量更新：

           修改错误代码！再次提交！

7.git 分支操作

  1. 主干开发

    • Google和Facebook等公司使用的就是主干开发模式
    • 要求开发人员能力较强，代码质量较高，不会反复修改！
    • 主干开发也适合Java，Java微服务服务开发， 每人负责一个服务，冲突率不高！

  2. Github Flow

    • 开发新的功能的时候创建新分支，完善以后，合并到主分支即可！
    • 这样的流程敏捷度较好，容错度较高！
    • 同时也可以配置code review，控制代码质量！
